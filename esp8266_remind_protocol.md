## 文档说明

本文档面向 **ESP8266 模块**开发同学，说明 STM32 主控在「闹钟响铃后超过 30 秒仍未按下取药确认键」时，经串口**上行**发送的数据包格式，便于在 ESP8266 侧解析并在 Web 页面上提示 **「请及时取走药物」**。

下行协议（ESP8266 → STM32 的心跳包、闹钟事件包）仍以项目根目录 [`readme.md`](readme.md) 为准。本文仅补充 **STM32 → ESP8266** 的提醒类指令。

---

## 业务背景（与主控联动逻辑）

1. ESP8266 在闹钟到达时，按现有协议发送带 `EVENT:RINGx` 的数据帧（例如 `EVENT:RING1`）。
2. STM32 收到后：舵机转到 90°，OLED 显示「请取走药物」，并开始 **30 秒**计时。
3. **30 秒内**用户按下确认键：舵机回 0°，OLED 显示「药物已取走」，**不会**向 ESP8266 发送本文所述数据包。
4. **超过 30 秒**仍未按键：STM32 **发送一次**取药超时提醒包（本文重点），OLED 显示「请及时取走药物」；此后用户再按键，主控仍只更新本地 OLED/舵机，**不会重复发送**同一轮提醒包。

## 通信参数

| 项目 | 值 |
|------|-----|
| 波特率 | **9600** |
| 数据位 | 8 |
| 校验位 | 无 |
| 停止位 | 1 |
| 物理连接 | STM32 **USART1 TX (PA9)** → ESP8266 **RX**；共 GND |
| 帧结束符 | `\r\n`（CRLF） |

---

## 取药超时提醒包（STM32 → ESP8266）

### 触发条件

- 已收到本轮闹钟的 `EVENT:RINGx` 事件；
- 自该事件起 **连续 30 秒**内未检测到取药确认按键；
- **每轮闹钟仅发送 1 次**（不会每 2 秒重复发送）。

### 数据包格式

```
STM32:CMD:REMIND_TAKE_MED,SLOT:x\r\n
```

| 字段 | 说明 |
|------|------|
| `STM32:` | 固定前缀，表示数据来自主控（与 ESP8266 下发的 `TIME:` 区分） |
| `CMD:REMIND_TAKE_MED` | 命令字：**取药超时提醒**，请在 Web 端映射为「请及时取走药物」 |
| `SLOT:x` | 闹钟槽位，`x` 为 `1` / `2` / `3`，与下行 `EVENT:RING1/2/3` 对应；若主控未能解析槽位则为 `0` |
| `\r\n` | 行结束，便于按行读取 |

### 示例

闹钟 1 触发且超时未取药：

```
STM32:CMD:REMIND_TAKE_MED,SLOT:1
```

闹钟 2 触发且超时未取药：

```
STM32:CMD:REMIND_TAKE_MED,SLOT:2
```

槽位未知时（兜底）：

```
STM32:CMD:REMIND_TAKE_MED,SLOT:0
```

## ESP8266 侧解析建议

### 识别方式

在串口接收缓冲中按**行**解析（以 `\n` 或 `\r\n` 断帧），满足以下**全部**条件即视为取药超时提醒：

1. 行内包含子串 `STM32:CMD:REMIND_TAKE_MED`；
2. （可选）进一步解析 `SLOT:` 后的数字，用于 Web 上区分是哪一个闹钟。

**不推荐**仅匹配 `REMIND` 或 `TAKE_MED` 等过短关键字，以免误触发。

### Arduino 参考代码（片段）

```cpp
void onSerialLine(String line) {
  line.trim();
  if (line.indexOf("STM32:CMD:REMIND_TAKE_MED") >= 0) {
  int slot = 0;
  int pos = line.indexOf("SLOT:");
  if (pos >= 0 && pos + 5 < (int)line.length()) {
    slot = line.charAt(pos + 5) - '0';  // '1'..'3' 或 '0'
  }
  // TODO: 更新 Web 状态，例如：
  // remindTakeMed = true;
  // remindSlot = slot;
  // 网页显示：「请及时取走药物」
  }
}
```

### Web 页面建议

| 收到数据包后 | 建议 UI 行为 |
|--------------|--------------|
| 解析到 `REMIND_TAKE_MED` | 弹出或常驻提示：**请及时取走药物** |
| 存在有效 `SLOT:1~3` | 可附加文案，如「闹钟 A1 已响铃，请尽快取药」（与现有 A1/A2/A3 命名一致即可） |
| 用户已在主控按键取药 | 本包不会出现；若需 Web 同步「已取走」，需另行约定（当前主控**不会**上行「已取走」包） |

## 与现有下行协议的对照

| 方向 | 示例 | 含义 |
|------|------|------|
| ESP8266 → STM32 | `TIME:08:00:01,...,EVENT:RING1` | 闹钟 1 触发 |
| STM32 → ESP8266 | `STM32:CMD:REMIND_TAKE_MED,SLOT:1` | 闹钟 1 响铃后 30 s 仍未确认取药 |

## 注意事项

1. **波特率必须为 9600**，与 [`readme.md`](readme.md) 中下行协议一致。
2. 提醒包为 **事件型单发**，请勿假设会周期性重复；Web 侧收到后应**保持提示**直到用户刷新或手动关闭（产品逻辑自定）。
3. 主控在发送提醒包时**不会阻塞** Web 服务；ESP8266 应在 `loop()` 中持续 `Serial.available()` 读取，避免软串口/硬件串口缓冲溢出丢包。
4. 若联调时收不到包，请确认：**STM32 TX 接 ESP8266 RX**、共地，且测试场景确实「闹钟已触发 + 30 s 内未按 PC2 确认键」。
5. OLED 上的「请及时取走药物」由 STM32 本地显示；Web 文案需 ESP8266 根据本协议**自行更新**，两边文案保持一致即可。

## 联调检查清单

- [ ] ESP8266 串口波特率 9600
- [ ] 能正常收到下行心跳 `TIME:...`
- [ ] 闹钟触发后，主控舵机动作正常
- [ ] **故意 30 s 内不按确认键**，串口监视器出现 `STM32:CMD:REMIND_TAKE_MED,SLOT:x`
- [ ] Web 页出现「请及时取走药物」
- [ ] 30 s 内按键取药时，**不应**出现上述上行包

如有字段扩展需求（例如增加时间戳 `TIME:`），请与 STM32 主控同学协商后再改协议，避免单边变更导致联调失败。
