## 核心功能

- **精准授时**：内置 NTP 客户端，直连腾讯时间服务器，自动同步标准时间。
- **移动端 Web UI**：内嵌响应式网页，全中文界面，支持类似 iOS 的卡片与滑动开关设计。
- **掉电记忆 (EEPROM)**：闹钟时间与开关状态自动保存至内部 Flash，断电重启不丢失配置。
- **非阻塞异步通信**：Web 服务与串口输出采用非阻塞设计，确保系统运行流畅不断联。
- **标准串口协议**：将复杂的时间数据转化为固定格式的纯英文字符串，主控端通过简单的 `strtok()` 即可高效解析。

## 快速上手

1. 打开代码文件，找到以下行，修改为你自己的 **2.4GHz** 手机热点或路由器 Wi-Fi 账号密码：
   ```cpp
   const char* ssid     = "你的WiFi名称";
   const char* password = "你的WiFi密码";
   ```
   
2. 将代码烧录至 ESP8266 开发板。
3. 打开 Arduino 串口监视器（**波特率设置为 9600**）。
4. 等待 ESP8266 连接网络，串口会打印出类似 `[WIFI] IP Address: xxx.xxx.xxx.xxx` 的信息。
5. 手机连接同一个 Wi-Fi，在浏览器中输入该 IP 地址，即可进入「闹钟控制中心」。

## 串口通信协议 (UART Protocol)

ESP8266 每约 **2 秒**会向串口发送一帧完整的数据包，波特率为 **9600**。

### 1. 心跳数据包格式

```
TIME:14:30:05,A1:08:00:1,A2:--:--:0,A3:15:45:0
```

- `TIME:xx:xx:xx`：当前网络时间。
- `A1/A2/A3`：代表三个闹钟槽位。
- `xx:xx:1` 或 `--:--:0`：设定时间。最后的 `1` 代表开启，`0` 代表关闭或未设置。

### 2. 闹钟触发事件包格式

当某个开启的闹钟时间到达时（精确到分钟），数据帧末尾会附加 `EVENT:RINGx`，**该事件会在当前分钟的前 2 秒内触发**（即发送一次带有 EVENT 的帧）。

Plaintext

```
TIME:08:00:01,A1:08:00:1,A2:--:--:0,A3:15:45:0,EVENT:RING1
```

*主控设备只需检测字符串中是否包含 `EVENT:RING` 即可触发蜂鸣器或其他硬件动作。*

## 注意事项 (Precautions)

1. **Wi-Fi 频段限制 (极度重要)**：

    ESP8266 硬件**仅支持 2.4GHz Wi-Fi**（IEEE 802.11 b/g/n），不支持 5GHz 频段。如果使用手机热点，请务必在手机热点设置中开启“最大兼容性”或手动将频段切换为 2.4GHz，否则设备将无限报错或假死。

2. **波特率对齐**：

    本程序的串口通信波特率已设定为 `9600`。在查看调试信息或与 STM32 进行连线通信时，主控端的接收波特率必须严格设置为 `9600`，否则会出现乱码。

3. **初次启动的 EEPROM 格式化**：

    代码会在首次运行时自动检测 Flash 并进行格式化（通过读取第 100 字节的 `0xAA` 校验位）。如果你后续更换了其他程序又刷回本程序，且发现设定的时间是乱码，可以通过擦除整个 Flash 重新烧录来重置 EEPROM。

4. **NTP 授时阻塞问题**：

    本项目默认使用腾讯的 NTP 服务器（`ntp.tencent.com`）。在极少数企业网络或校园网环境下，UDP 123 端口可能会被防火墙屏蔽导致无法获取时间。此时建议使用手机流量开热点进行测试。

5. **硬件供电**：

    ESP8266 在连接 Wi-Fi 的瞬间会有较大的电流峰值（约 300-400mA）。如果与其他设备共用电源，请确保 3.3V 供电芯片（如 AMS1117-3.3）能够提供足够的电流余量，避免因瞬间电压拉低导致频繁重启。
